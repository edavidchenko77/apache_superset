## Описание проекта

Полный цикл аналитики данных для анализа процессов президентства в США:
- Извлечение данных с Kaggle
- Загрузка в PostgreSQL
- Создание аналитической витрины
- Визуализация в Apache Superset


##  Исследуемые метрики бизнес-процесса
- **Цена (Price)** — стоимость объекта недвижимости.
- **Тип недвижимости (Type)** — классификация объектов (house, unit, townhouse и др.).
- **Количество комнат (Rooms)** — размер жилья в комнатах.
- **Удалённость от центра (Distance)** — расстояние до центрального делового района Мельбурна.
- **Средняя цена по субурбу (Average Price by Suburb)** — позволяет сравнивать районы.
- **Количество объектов по типу и району (Count by Type/Suburb)** — для оценки плотности рынка.
 
## Выводы по работе


В ходе выполнения работы был освоен полный цикл аналитики данных с использованием современных инструментов ETL и визуализации. Созданный дата-конвейер на Apache Airflow позволяет автоматически извлекать данные с внешнего источника (Kaggle), загружать их в PostgreSQL и трансформировать для построения аналитической витрины.
На основе подготовленных данных был разработан дашборд в Apache Superset, включающий все требуемые типы визуализаций: индикаторы, столбчатые и круговые диаграммы, комбинированные и линейные графики. Дашборд предоставляет возможность анализа ключевых метрик рынка недвижимости Мельбурна, таких как средняя цена, распределение по типу жилья, количество комнат и удалённость от центра.

Выводы построенные на полученноми дашборде.
Наиболее дорогие районы находятся ближе к центральному деловому району .
Объекты типа house в среднем дороже юнитов и таунхаусов.
Существует прямая зависимость между количеством комнат и ценой недвижимости.
